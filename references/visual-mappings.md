# Structured Visual-to-Parameter Mappings

## How to use this document

During jewelry image analysis, map each observed visual feature to the **31 jewelry parameters** using the tables below.

Each entry follows this shape:
- **Visual cue** — what to look for in images/text
- **Parameter mapping** — which parameter is influenced
- **Allowed values** — restricted to the recognized option set(s) where applicable
- **Evidence rules / priority** — how to resolve ambiguity

---

## Inference Rules

1. **Preferred source level:** If a diagram exists in the GIA guide for the observed feature, prefer it over text-only options from the Excel catalog.
2. **Composite decisions:** When multiple features coexist, produce **all applicable parameter values** in priority order.
3. **Null protocol:** If a feature cannot be visually determined, use `null`.
4. **Metal ambiguity:** When karat cannot be established from visual cues alone, use the lowest specific Metal Type consistent with the observed karat-mapped features, or `null` if none apply.
5. **Gender inference:** Default to `null` unless visual style/size strongly suggests Baby/Gents/Ladies.
6. **Parameter order:** Always return parameters in the canonical 31-field order defined in SKILL.md.
7. **Parse explicit mentions first:** Any product title/text mentioning a term that maps cleanly to a GIA term overrides visual inference.
